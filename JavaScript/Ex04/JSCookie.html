The first browser to use cookie functionality was Netscape. In Computer Science parlance, a cookie is an opaque piece of data held by an intermediary. In other words it is a small data file on your hard disk (intermediary) that is used by web domains to store some data on your machine

The typical usage of cookies is for holding user information to personalise a website for the user. This can be information such as name, color preferences, etc.

It is generally a bad idea to store any personal information (including name) in a cookie. Cookies do present a risk to users if developers do not adhere to strict data protection controls.

The latest form of attack (2013) is 'JavaScript Cookie Bomb'. It works like this:

  • Compromised Site, or mal-intentioned site puts/creates Cookie on your machine
  • Cookie contains iframe code that renders into the webpage. The iframe content is whatever hacker wants it to be, typically an advert of some kind. It could also be an exploit kit or other malware.

A good resource for Cookie information is: cookiecentral.com

Some believe that the cookie has had its day, and will be replaced by local storage systems.
